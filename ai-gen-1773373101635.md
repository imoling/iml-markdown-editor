## 1\. 项目概述

### 1.1 产品名称

**智会助手** (暂定名)

### 1.2 核心问题与机遇

现代会议普遍存在效率低下、信息分散、行动项追踪困难等问题。与会者常常需要同时处理**记录、理解、参与、跟进**多项任务，导致注意力分散，会议价值衰减。我们的目标是打造一个无缝融入会议流程的智能助 >\*\*AI 总结\*\*：当前会议管理实践中，普遍存在任务记录、信息理解、实时参与及后续跟进等多线程工作并行处理的困境。与会者往往被迫在倾听讨论、消化观点、记录要点与思考行动项之间频繁切换，这种认知负荷的持续分散不仅加剧了与会者的精神疲劳，更使得会议的深层价值——如战略对齐、创意碰撞与共识凝聚——在碎片化的注意力中被悄然稀释。为此，我们亟需重新审视会议流程，探索能够整合并优化这些环节的解决方案。，我们致力于开发一款智能会议助手，其核心目标是深度融入现有会议流程，通过智能化支持减轻认知负荷，使团队能更聚焦于会议本身的沟通与决策，从而提升会议效率与产出价值。 手，将与会者从繁琐的行政事务中解放出来，聚焦于核心的沟通与决策。

### 1.3 核心目标与愿景

**愿景**：成为每一场高效会议的“默认基础设施”，让会议从成本中心转化为价值创造中心。 **核心目标**：

*   **降本提效**：自动化会议前后80%的行政与信息整理工作。
    
*   **知识沉淀**：将会议中的隐性知识（讨论、决策上下文）转化为可搜索、可复用的组织资产。
    
*   **驱动执行**：清晰定义并追踪行动项，确保会议决策落地。
    

### 1.4 目标用户画像

1.  **会议组织者/主持人**：需要安排会议、整理纪要、分发任务。
    
2.  **核心参会者**：需要专注讨论，同时确保自己的观点和任务被准确记录。
    
3.  **列席者/后加入者**：需要快速了解会议背景、讨论过程和结论。
    
4.  **团队管理者**：需要洞察团队讨论质量、任务负荷与执行情况。

## 2\. 核心功能描述

### 2.1 会前智能准备

*   **2.1.1 一键预约与同步**：连接企业日历（Google Calendar, Outlook），输入会议主题后，自动建议时间、参会人，并生成包含议程模板的会议邀请。
    
*   **2.1.2 智能议程生成**：根据历史同类型会议或用户输入的几个关键词，自动生成结构化议程草案（如：目标、背景资料、讨论议题、待决议项）。
    
*   **2.1.3 背景信息推送**：在会议开始前，自动向参会者推送本次会议相关的历史纪要、待决行动项、共享文档链接。
    

### 2.2 会中实时辅助

*   **2.2.1 高精度语音转文字**：支持多说话人识别，实时转录会议对话，并标识发言人。
    
*   **2.2.2 关键信息自动捕捉**：
    
    *   **行动项识别**：自动检测“我来负责”、“下周五前完成”等承诺性语句，并提取为待办事项（任务描述、负责人、截止时间）。
        
    *   **决策点识别**：自动标记“那么我们就决定...”、“同意方案A”等决策结论。
        
    *   **问题与待办标记**：用户可通过语音指令（如“标记一个问题”）或界面快捷按钮，手动标记当前讨论内容。
        
*   **2.2.3 实时摘要与大纲**：在侧边栏动态生成会议讨论的要点摘要和结构化大纲，帮助用户把握进度。
    

### 2.3 会后自动生成与分发

*   **2.3.1 一键生成智能纪要**：
    
    *   自动生成结构清晰的会议纪要，包含：基本信息、参会人、议程回顾、讨论要点、**决策列表**、**行动项清单**（高亮显示）。
        
    *   支持用户在线编辑、高亮重点内容。
        
*   **2.3.2 多渠道智能分发**：
    
    *   自动通过邮件将纪要给全体参会者及指定抄送人。
        
    *   自动将**行动项**同步至团队协作工具（如 Slack, Teams, 飞书，钉钉）的指定群组或项目管理系统（如 Jira, Asana, Trello）。
        
*   **2.3.3 知识库自动归档**：所有纪要按照项目、团队、标签自动归档，形成可全文搜索的会议知识库。

## 3\. 用户交互细节

### 3.1 核心交互流程

1.  **会前**：用户在日历中点击“使用智会助手安排”，填写主题 -> 智能生成议程 -> 调整 -> 发送邀请。
    
2.  **会中**：
    
    *   **桌面端/Web端**：主界面为实时转录文本，右侧为动态大纲和行动项面板。用户可点击任何转录文本进行“标记为决策”、“标记为问题”。
        
    *   **移动端/仅音频接入**：用户拨入专属会议号码，助手自动接入并开始转录。通过语音指令（“智会助手，记录一个行动项：我下周修复这个BUG”）进行交互。
        
3.  **会后**：会议结束5分钟内，用户收到邮件“您的会议纪要已就绪”，点击链接进入编辑/确认页面，一键点击“分发并归档”。
    

### 3.2 UI/UX 关键点

*   **极简介入**：作为虚拟参会人加入会议，无需所有参会人安装特定App。
    
*   **状态可视**：会中界面清晰展示录音状态、转录准确度、已识别的行动项数量。
    
*   **编辑友好**：纪要编辑界面类似Notion，支持块级编辑，对自动提取的内容（行动项、决策）有特殊视觉组件。
    
*   **隐私提示**：首次使用及录音开始前有明确的隐私提示，告知用户数据如何处理。

## 4\. 技术栈建议 (适配 Vibe Coding 的简洁方案)

为快速验证核心价值，建议采用现代、高效、全栈JavaScript的技术方案。

| 层级 | 技术栈 | 说明 |
| --- | --- | --- |
| 前端 | Next.js 14 (App Router) + Tailwind CSS + Shadcn/ui | 服务端渲染，优秀的开发体验，可快速构建美观、响应式的管理界面。Shadcn/ui 提供高质量的组件。 |
| 实时交互 | Socket.io 或 Pusher | 用于会中实时转录文本流式推送到前端界面。 |
| 后端/API | Next.js API Routes 或 NestJS | Next.js API Routes 适合全栈快速开发。若逻辑复杂，NestJS 提供更清晰的结构。 |
| AI/核心处理 | OpenAI Whisper API / Azure Speech SDK | 语音转文字。OpenAI GPT-4 Turbo API |
| 数据库 | PostgreSQL (Vercel Postgres, Supabase, Neon) | 关系型数据，存储用户、会议、纪要、行动项。 |
| 任务队列 | Bull (基于Redis) 或 Inngest | 处理异步任务，如会后邮件发送、数据归档、AI处理。 |
| 存储 | AWS S3 或 Vercel Blob | 存储会议录音文件（如需保留）。 |
| 部署 | Vercel (前端) + Railway / Fly.io (后端服务) | 面向开发者的极简部署体验，完美适配此技术栈。 |

## 5\. 验证计划

*   **核心功能**：会议接入、语音转文字、手动创建行动项、基础纪要编辑与邮件发送。
    
*   **成功指标**：
    
    1.  **可用性**：10个内部团队，每周至少使用3次，持续4周。
        
    2.  **核心任务完成率**：90%的会议能成功生成并发送一份“可用”的纪要（用户无需重写）。
        
    3.  **用户满意度**：NPS > 50。
        

### 5.2 第二阶段：价值深化验证

*   **核心功能**：上线AI自动提取行动项与决策、知识库搜索、第三方工具集成。
    
*   **成功指标**：
    
    1.  **效率提升**：用户反馈“准备与整理会议时间”平均减少超过60%。
        
    2.  **行动项落地率**：通过系统派发的行动项，在截止日期前的完成率提升至80%（需定义基线）。
        
    3.  **留存与扩张**：MVP用户月活留存率 > 70%，并有20%的团队主动邀请其他团队使用。
        

### 5.3 长期成功定义

*   **数据价值**：会议知识库成为团队新成员入职培训、项目复盘的重要信息来源。
